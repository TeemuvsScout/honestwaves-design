How the four pieces work together

Breadcrumb (escape hatch)

Always present at the top of the report. Every segment is clickable. At depth 4 (a single user), clicking Troy Hospital jumps you straight back to depth 1 — you don't have to undo each level. This is the single most important piece for keeping users oriented during deep dives.

KPI strip (scope confirmation)

The KPIs always reflect the current depth, not the company total. At "All locations" they show company-wide $14,247 lost. At "Troy Hospital → ER" they show $4,829 lost. This is what lets a department head answer "are my losses going up?" without doing the subtraction in their head.

Group-by pivot (lens swap)

Same data, different aggregation. At "All locations" you can group by Location (default) or by Department or by Model or by User. At "Troy → ER" the pivot offers Locker / User / Model. Each grouping is a different angle on the same scope. Keep the breadcrumb intact when the pivot changes.

Drill table (the main work surface)

Every row is clickable. The trailing column shows the dollar total — the user picks the biggest number and clicks. The breadcrumb extends, KPIs update, table refreshes one level deeper. Repeat until you're at a single incident.

Entry points (so users don't always start at the top)

  • Search — top-right of every report. Type "Pink8" → land on the Pink8 detail with breadcrumb All › Troy › ER › Pink8.
  • Deep links from other widgets — clicking "Mark Rivera" on the dashboard high-risk-users tile lands you directly at All › Troy › ER › Pink8 › Mark Rivera. Same screen, deeper start.
  • Saved views — leadership can save "Troy ER · This quarter" as a one-click bookmark in the nav.
  • URL state — every drill level updates the URL. Sharing a link to ?loc=troy&dept=er opens the report at the right depth.

Reuse

The same four-part pattern (breadcrumb + KPI + pivot + drill-table) applies to every aggregate report in Asset Accounting — Loss & damage, Inventory value, Repair tracker. Build the skeleton once as a single component, feed it different metric configs per report, and you get five drill-down reports for the price of one.
