PrimeVue + Tailwind mapping

Build this in PrimeVue (Aura unstyled preset) + Tailwind. Map every token here to a custom Tailwind theme value, then use PrimeVue's pt (pass-through) props to override component classes. Key component swaps from your current implementation:
  • DataTable — for Devices / Users / Reports. Compact size, hover row highlight.
  • Drawer — for Edit User / Edit Device / Branding asset (replaces today's bottom-panel slide).
  • Dialog — for Add modals + Delete confirmations.
  • Tag (custom severity) — for every state pill in this prototype.
  • InputGroup + IconField — for search inputs everywhere.
  • Menu (popover) — for avatar dropdown, row context menus.
  • Toast + ConfirmDialog services — the two patterns from the Confirmations & toasts copy-audit tab.
  • Skeleton — replace the spinning circles I saw on the live dashboard with skeleton rectangles per widget.

Density rules

Ops-first: smaller row heights, smaller font in tables, but never below 12px for primary content. Padding is for grouping, not for breathing room.
  • Sidebar item height: 32px (was ~40px)
  • Table row height: 36px compact (was ~52px)
  • Topbar height: 52px (was ~64px)
  • State pill height: 20px (uniform; was inconsistent 22–28px)
  • Card padding: 16/20px (was 20/24px+)
