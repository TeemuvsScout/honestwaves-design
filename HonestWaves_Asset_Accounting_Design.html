HonestWaves Dashboard — Asset Accounting (Design Prototype)

A proposed new top-level section in the left nav, between Reports and Settings. Click the items inside the prototype to navigate. Mock data uses your client's two device types (iPhone 16e for clinical staff, AiRista badge for security/maintenance) and realistic department + location names.

Why a top-level section

The accounting story (who lost what, how much, where) needs a single home where finance, ops leadership, and department heads land. Burying it inside Reports loses that surface. Putting it in the left nav signals to the customer that asset accountability is a first-class concern.

Five sub-pages

  • Overview — KPI dashboard for the C-suite glance.
  • Loss & damage reports — the workhorse, pivotable by user / department / location / model.
  • Asset inventory value — what's in each kiosk right now, by location.
  • Repair & replacement tracker — the new bit; handles partial damage cases (iPhone screen, charging port) separately from total loss.
  • Thresholds & rules — where customers configure "user has lost ≥3 → supervisor approval".

Inline placements (also shown at the bottom of this file)

Asset Accounting is a hub and a set of inline placements: Device Model gets the cost + repair components, Users get a lifetime $ total, Kiosks get a Total Bay Value badge, the Dashboard gets a Loss Exposure tile. Scroll past the prototype to see those.

Thresholds & rules

Automated actions when a user, department, or device class crosses a loss threshold. Rules apply across all kiosks for this tenant.

User-level thresholds

Triggered when an individual user crosses a defined limit

Block checkout after 3 lost devices
User cannot check out new devices until a supervisor signs in at the kiosk. Currently affects Mark Rivera.
1 user blocked · 4 users at warning · Edit → Threshold 3 · Window 90 days
Require supervisor approval if user's loss > $500
Lifetime $ loss threshold. Approval required at next checkout. Supervisor receives push notification.
2 users currently in this state · Edit → Threshold $500 · Window: All time
Email warning to user at 2 incidents
Sends "Asset loss warning — next incident blocks your checkout privileges" with a link to lost-asset policy.
4 users warned this quarter · Edit → Threshold 2 · Window 90 days
Auto-overdue threshold
Mark a checkout overdue 1 hour past expected return time. Fires the existing overdue notification flow.
Default value · Edit → 1 hour
Photo at return for high-value devices (> $500)
Kiosk camera captures a photo of the device at return for any model with replacement cost over the threshold. Stored against the event for damage disputes.
Edit → Threshold $500 · Applies to iPhone 16e, Pixel 8 Pro, ChromeBook X360

Department-level thresholds

Aggregate loss across a whole team

Email department head when quarterly loss > $3,000
Sends a digest email to the department head listing top loss-prone users and recent incidents. Troy ER and Troy ICU currently exceeding.
2 departments triggered this quarter · Edit → Threshold $3,000 · Cadence: Quarterly
Chargeback to department on loss
Each loss event creates a ledger entry against the user's department. Exported to your finance system monthly.
23 chargebacks this quarter · Edit → Integration: SAP via webhook
