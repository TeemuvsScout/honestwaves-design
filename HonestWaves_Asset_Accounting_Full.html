Asset Accounting — complete design system

Every screen, sub-page, and modal for the proposed Asset Accounting section, in one prototype. Click any tab in the left nav to switch main sections; tabs within each page swap sub-views. The bottom of this file is a gallery showing every modal/popup state side by side so your designer can see them all without triggering each one.

5 main pagesOverview · Loss & damage · Inventory value · Repair tracker · Rules

Mock dataiPhone 16e + AiRista, Troy + Butterworth, real-ish names

What's clickable

  • Left nav buttons switch between the 5 main pages.
  • Sub-tabs at the top of each page swap between the page's sub-views (e.g. By User / By Department / By Location / By Model).
  • Pivot toggles re-aggregate the rows.
  • Switches/toggles flip state.
  • Modals at the bottom of the file are static screenshots — they're shown stacked so you can review every popup in one scroll.

Thresholds & rules

Automated actions when a user, department, or device class crosses a defined limit.

User-level rules

Block checkout after 3 lost devices
User cannot check out new devices until a supervisor signs in. Currently affects Mark Rivera.
1 blocked · 4 at warning · Threshold 3 · Window 90 days
Require supervisor approval if loss > $500
Lifetime $ loss threshold. Approval required at next checkout.
2 users in this state · Threshold $500 · All time
Email warning to user at 2 incidents
Sends "next incident blocks your checkout privileges" with policy link.
4 users warned this quarter · Threshold 2 · Window 90 days
Photo at return for high-value devices (> $500)
Kiosk camera captures device photo at return. Stored for damage disputes.
Applies to iPhone 16e, Pixel 8 Pro, ChromeBook X360

Department-level rules

Email department head when quarterly loss > $3,000
Digest email with top loss-prone users and recent incidents. Troy ER and Troy ICU currently exceeding.
2 departments triggered · Threshold $3,000 · Quarterly
Chargeback to department on loss
Each loss event creates a ledger entry against the user's department. Exported to finance monthly.
23 chargebacks this Q · Integration: SAP via webhook
