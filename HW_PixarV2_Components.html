Button

Use PrimeVue's <Button> with built-in severity props. No custom button class needed.

<Button label="Primary" />
<Button label="Secondary" severity="secondary" />
<Button label="Success" severity="success" />
<Button label="Danger" severity="danger" />
<Button label="Outlined" outlined />
<Button label="Text" text />
<Button label="Add widget" icon="pi pi-plus" />
<Button icon="pi pi-bell" rounded text />
Tailwind add-ons: h-[30px] for compact density, w-full for block buttons. No need to override colors — severity prop handles it.

Tag (a.k.a. pill / status badge)

PrimeVue's <Tag> with severity. Same pattern as Button.

Available Overdue Lost In use Deactivated ● Active NEW 
<Tag value="Available" severity="success" />
<Tag value="Overdue"   severity="warning" />
<Tag value="Lost"      severity="danger" />
<Tag value="In use"    severity="info" />
<Tag value="Active"    severity="success" rounded />

<!-- Pixar accent (NEW badges) — Tailwind class override -->
<Tag value="NEW" :pt="{ root: { class: 'bg-pixar-soft text-pixar' } }" />
The pass-through API (:pt) is how you customize PrimeVue components with Tailwind classes without ejecting from the library.

KPI tile (the one custom component)

PrimeVue doesn't ship a KPI/stat component. Build a thin wrapper around <Card> with Tailwind utilities — ~30 lines.

<!-- KpiTile.vue -->
<template>
  <Card>
    <template #content>
      <div class="text-[10px] font-bold uppercase tracking-wider
                  text-slate-500 flex items-center gap-1.5 mb-1">
        <span class="w-1.5 h-1.5 rounded-full"
              :class="dotColor"></span>
        {{ label }}
      </div>
      <div class="text-3xl font-semibold tabular-nums tracking-tight"
           :class="valueColor">
        {{ value }}
      </div>
      <div class="text-xs text-slate-500 mt-0.5">
        <slot />
      </div>
    </template>
  </Card>
</template>

<script setup>
defineProps<{
  label: string;
  value: string | number;
  severity?: 'brand' | 'success' | 'warning' | 'danger';
}>();
</script>

That's the whole system.

Every Pixar v.1 pattern maps to a PrimeVue component +/- Tailwind utilities. The only custom component you have to build is KpiTile (30 lines around a <Card>). Everything else is already in your component library — no reinventing wheels.
